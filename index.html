01 Game Overview

Alien Cuisine Rush follows four brave astronaut chefs on a mission — to feed the galaxy. Venturing to unknown planets, they gather exotic alien ingredients and set up their cosmic restaurant at the most popular alien hotspots in the universe!

On open days, the crew cooks and serves famished aliens desperate for a slice of Klumvorian pizza or a sought-after Blumferfish burger. On closing days, the heroes scour even more distant worlds for the rarest ingredients in the cosmos.

02 Game Goal

Cook and serve food to alien customers before the timer runs out. Each correct order earns points — but mess up too many times and the restaurant closes for good!

✓ Win Condition
Serve all aliens and earn the highest score before time runs out.
✗ Lose Condition
Make 3 mistakes (wrong orders or slow service) and lose all health.

04 Game Mechanics & Logic

Main Actions:

  • Gather exotic ingredients from around the map
  • Prep and cook alien dishes at the kitchen station
  • Serve completed meals to waiting alien customers
  • Manage time across multiple orders at once

Variables:

Variable Name Starting Value What It Does
score0Tracks total points earned from serving aliens
health3Number of mistakes allowed before game over
timeRemaining60 secondsCountdown timer for the current serving shift
aliensServed0Counts how many alien customers have been fed
currentOrdernullStores which dish the current alien wants

If / Then Rules:

  • If the player cooks the food correctly → add 10 points to score
  • If the player serves the food to the alien in time → add 15 bonus points
  • If the player takes too long or cooks the food wrong → lose 1 health
  • If health equals 0 → the game ends immediately (lose)
  • If timeRemaining equals 0 → check score — win or lose based on goal
  • If all aliens are served → the player wins the level

05 Game Loop

These steps repeat continuously while the game is running:

  1. Check for player input — did the player press a key or button?
  2. Move the player character based on the input received
  3. Check if a new alien customer has appeared with an order
  4. Player collects ingredients and prepares the requested dish
  5. Player serves the completed dish to the waiting alien
  6. Update score, health, and aliensServed based on the result
  7. Check win and loss conditions (health = 0, time = 0, all served)
  8. Repeat until the game ends

06 Playtesting & Changes

Changes made after testing the game with real players:

⚠ Problem Found

Players said the game was too easy — they could serve all aliens without losing any health.

✓ Changes Made

Added more alien customers per round and introduced a faster difficulty mode. Alien patience timers now decrease as the level progresses.

⚠ Problem Found

It wasn't clear which dish each alien wanted — players kept guessing wrong.

✓ Changes Made

Added a visual speech bubble above each alien showing a picture of their desired food item.

08 Designer Reflection

The hardest part of building Alien Cuisine Rush was designing the time and scoring system so the game felt fair but challenging. Early versions were too easy, which playtesting quickly revealed — we adjusted alien patience timers and added more customers per level to fix it.

This project taught us how important it is to actually test your game with other people. Things that seemed obvious to us as designers were confusing to players, like the food order system. Adding visual speech bubbles was a simple fix that made a big difference.

Our game uses core programming concepts — variables to track score, health, and time; if/then logic to check win and lose conditions; and loops to keep the game running every frame. These ideas show up in almost every game ever made, so learning them here is a solid foundation.
